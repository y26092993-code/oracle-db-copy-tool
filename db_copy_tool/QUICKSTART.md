# Oracle DB オブジェクトコピーツール - クイックスタートガイド

## 基本的な使い方

### 1. アプリケーションを起動

```bash
python db_copy_gui.py
```

または exe ファイルをダブルクリック

### 2. 接続設定

#### tnsnames.ora が利用可能な場合

1. 「接続設定」タブを開く
2. ソースDB欄の **TNS エントリ** ドロップダウンから接続先を選択
3. **ユーザー名** と **パスワード** を入力
4. ターゲットDB欄も同様に設定
5. 「接続テスト」ボタンをクリックして確認

#### 手動入力の場合

1. 「接続設定」タブを開く
2. 以下を入力:
   - ホスト: `192.168.1.100`
   - ポート: `1521`
   - サービス名/SID: `ORCL`
   - ユーザー名: `scott`
   - パスワード: `tiger`
3. 「接続テスト」ボタンをクリックして確認

### 3. オブジェクトを選択してコピー

#### パターンA: 全てのテーブルをコピー

1. 「オブジェクト選択」タブを開く
2. 「テーブル」にチェック
3. ワイルドカード欄は **空白のまま**
4. 「一覧を更新」で確認
5. 「実行とログ」タブで「コピー実行」

#### パターンB: USER_で始まるテーブルのみコピー

1. 「オブジェクト選択」タブを開く
2. 「テーブル」にチェック
3. ワイルドカード欄に `USER_%` と入力
4. 「一覧を更新」で確認（USER_で始まるものだけ表示される）
5. 「実行とログ」タブで「コピー実行」

#### パターンC: 複数のパターンでコピー

1. 「オブジェクト選択」タブを開く
2. 必要なオブジェクトタイプにチェック
3. ワイルドカード欄に `USER_%, CUSTOMER_%, ORDER_%` と入力
4. 「一覧を更新」で確認
5. 「実行とログ」タブで「コピー実行」

## コピーオプション

### 「コピー前にターゲットのオブジェクトを削除 (DROP)」

- **テーブル/ビュー**: チェックで削除してからコピー
- **その他（プロシージャ等）**: このオプションにかかわらず常に削除

### 「エラーが発生しても続行」

エラーが発生しても処理を継続します。ログで詳細を確認できます。

### 「コピー前に差分を確認する」

実際のコピー前に、ソースと対象DB間のオブジェクト差分を表示します。本番関連のコピー時などで、意図しないオブジェクトがないか確認できます。※大量のオブジェクトをコピーする場合は、チェックを外すと高速に実行できます。

## 実行方法

### 1. ドライラン（検証）
1. 「オブジェクト選択」タブでオブジェクトを選択
2. 「実行とログ」タブで **「ドライラン」** ボタンをクリック
3. ログで「[DRY-RUN]」マークが付いて、実際には実行されていないことを確認
4. エラーが出ていないを確認後、本番実行を進める

### 2. 本番実行（進捗表示）
1. ドライランで検証後、「実行とログ」タブで **「コピー実行」** ボタンをクリック
2. プログレスバーと「●●個中 ●●個完了」テキストで進捗を確認
3. 大量のオブジェクト処理も進捗が見えるので安心

## パッケージ優先度

3つの新機能を詳細に説明します：

### 1. パッケージ優先コピー
オブジェクトを以下の優先順でコピーします。これにより、依存関係のエラーを防ぎます：

```
PACKAGE（パッケージ）
  ↓
SEQUENCE（シーケンス）
  ↓
PROCEDURE/FUNCTION（プロシージャ/ファンクション）
  ↓
TRIGGER（トリガー）
  ↓
VIEW/SYNONYM（ビュー/シノニム）
  ↓
TABLE（テーブル）
```

### 2. ドライランモード
実際のコピーを行わず、どのオブジェクトをどの順序でコピーするのかを確認できます：
- 本番環境へのコピー前に「予行演習」として使用
- エラーが発生しそうな対象を事前に検出
- ネットワーク遅延やDBの問題がないか確認

### 3. 進捗表示
大量のオブジェクト処理時にリアルタイムで進捗を表示：
- プログレスバーで視覚的に進捗を確認
- 「●●個中 ●●個完了（進捗率●●%）」のテキスト表示
- 何個処理中で、あと何個残っているかが一目瞭然

## ワイルドカードパターンの例（Oracle LIKE 演算子形式）

| パターン | 説明 | 例 |
|---------|----|-----|
| `USER_%` | USER_で始まる | USER_TABLE1, USER_VIEW1 |
| `%_TMP` | _TMPで終わる | DATA_TMP, BACKUP_TMP |
| `TEST%` | TESTで始まる | TESTDATA, TEST_PROC |
| `%TEST%` | TESTを含む | ATESTB, TEST1, MY_TEST |
| `DATA____` | DATA_の後に4文字 | DATA_2024, DATA_TEMP |
| `USER_%, TEMP_%` | 複数パターン | USER_XXX または TEMP_XXX |

## よくある質問

### Q1: データもコピーされますか？

A: いいえ、このツールは **オブジェクトの構造（DDL）のみ** をコピーします。テーブルのデータはコピーされません。データをコピーする場合は、Data Pump などを使用してください。

### Q2: パッケージをコピーすると INVALID になります

A: パッケージに依存関係がある場合、依存元のオブジェクトを先にコピーする必要があります。または、全てをコピーした後に `ALTER PACKAGE xxx COMPILE;` で再コンパイルしてください。

### Q3: ワイルドカードで除外することはできますか？

A: 現在の実装では、マッチするパターンのみ指定できます。除外したい場合は、コピーしたいオブジェクトのパターンを指定してください。

例: TEST_ で始まるものを除外したい場合
→ `PROD_%, USER_%` など、コピーしたいパターンを指定

### Q4: 大文字小文字は区別されますか？

A: いいえ、ワイルドカードパターンは **大文字小文字を区別しません**。`user_%` と `USER_%` は同じです。

### Q5: エラーが出てもコピーを続けられますか？

A: はい、「エラーが発生しても続行」オプションにチェックを入れると、一部のオブジェクトでエラーが発生しても処理を続行します。ログで詳細を確認できます。

### Q6: 大量のオブジェクトをコピーするときに差分確認を表示したくないのですが？

A: 「コピー前に差分を確認する」のチェックを外してください。差分確認ウィンドウが表示されず、すぐにコピーが開始されます。

### Q7: エラーが大量に発生したときに確認するには？

A: 実行完了後、「エラーのみ表示」ボタンをクリックすると、発生したエラーメッセージのみを一覧で表示できます。

### Q8: ドライランで何か問題が見つかったら？

A: ドライランで「[DRY-RUN]」マークが付いているので、実行されていません。ログを確認してコピーするオブジェクトを調整してから、本番実行してください。

### Q9: パッケージをコピーするとき、なぜパッケージを優先しているのですか？

A: パッケージに依存するプロシージャやビューがある場合、パッケージを先にコピーしないと「存在しない参照」エラーが発生します。優先度を設定することで、自動的に正しい順序でコピーできます。

## トラブルシューティング

### 接続できない

- ホスト、ポート、サービス名を確認
- ネットワーク接続を確認（ping コマンドなど）
- Oracle Listener が起動しているか確認
- ファイアウォールの設定を確認

### オブジェクトが見つからない

- ユーザーに SELECT 権限があるか確認
- スキーマが正しいか確認
- ワイルドカードパターンが正しいか確認

### コピーに失敗する

- ターゲットDBに CREATE 権限があるか確認
- 既存のオブジェクトがある場合は「DROP」オプションを有効に
- ログファイルで詳細なエラーを確認

## サポート

問題が解決しない場合は、ログファイル（`db_copy_tool_YYYYMMDD.log`）を確認して、開発チームに連絡してください。
